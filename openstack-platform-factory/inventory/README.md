# Inventory Model

The inventory layer represents the operational and infrastructure asset view of the platform.

It contains information retrieved from:

* CMDB systems
* Infrastructure discovery
* Cloud APIs
* Hypervisors
* Network devices
* Existing deployment inventories

The inventory is used by DAEDALUS to validate that the real platform matches the architectural intent defined in the HLD and implemented through the LLD.

---

# Philosophy

DAEDALUS separates:

## Architecture

Defined by the HLD.

Represents:

> What the platform should be.

---

## Inventory

Defined by the inventory layer.

Represents:

> What the platform currently is.

---

# Purpose

The inventory layer is used to:

* Validate architecture consistency
* Detect infrastructure drift
* Validate deployment readiness
* Generate deployment inventories
* Validate HA topology
* Validate networking consistency
* Validate storage topology
* Support governance controls

# Raw Inventory

The `raw/` directory contains data retrieved directly from external systems such as:

* CMDB platforms
* Cloud APIs
* Discovery tools
* Existing infrastructure inventories

Examples:

* Hypervisor inventory
* Network topology
* IP allocations
* Existing OpenStack nodes
* Storage nodes
* Security zones

The raw inventory is considered:

> External operational source of truth.

---

# Generated Inventory

The `generated/` directory contains inventories generated by DAEDALUS after validation and transformation.

Examples:

* Ansible inventories
* Terraform variable files
* Deployment topology files
* Host group definitions

Generated inventories are derived artifacts and should not be manually modified.

---

# Validation Model

DAEDALUS validates consistency between:

* HLD models
* LLD models
* Inventory data
* Governance constraints
* Identity requirements

Examples of validation checks:

* Multi-AZ consistency
* Controller quorum validation
* Storage replication validation
* Network segmentation validation
* Identity integration validation
* Capacity validation

# CMDB Integration

DAEDALUS supports integration with enterprise CMDB platforms.

The CMDB provides:

* Operational inventory
* Asset metadata
* Environment relationships
* Platform ownership
* Deployment topology information

DAEDALUS uses this information to validate architectural consistency before deployment or promotion.
